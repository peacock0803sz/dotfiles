# ドキュメント規則

- 日時記載禁止。更新日時等の記載は不要
- 簡潔性重視。工数/後方互換性の考慮は不要

## 記号・表記

- 句読点は前の文字が日本語なら「、。」、英数字等なら「,.」を使う
- 記号は可能な限り全角を使わず半角にする。句読点「、。」は除く
- 半角英数字の前後にスペースを入れない。日本語と英数字の間に余白を足さない
- 装飾目的の太字(`**`)は使わない。強調が本当に必要なときだけ残す
- カギ括弧「」を多用しない。引用や固有の呼称に必要なときだけ使い、それ以外は文脈に溶かす
- 丸括弧()を多用しない。補足は括弧に逃がさず本文の一文として組み込む
- コロン(：)は原則使わない。使う場合も直後に半角スペースを入れない。ラベル列挙はしない
- スラッシュ(／)や矢印(→)で概念を並列しない。文章として書く

## 文体

- 前置き宣言を入れない。「本記事では」「以下で解説します」のような宣言から書き出さず、いきなり本文として書く
- 締めの定型句を入れない。「参考になれば幸いです」のような決まり文句は使わない
- 情報を増やさない安全クッションを削る。「一般的に」「状況によって異なります」等は最小限の注意書きに圧縮する
- 抽象語だけで押し切らない。「重要」「効果的」「最適」等に頼らず、何がどうなるかを動詞中心で具体的に書く
- 同義語の言い換え連打を避け、1回で言い切る

## コメント規則 (コード内)

- 複雑なロジックのみコメント記載
- 内容は「何を」や「どうやって」ではなく「何故」を書くこと
- 日本語で(OSSリポジトリは英語)でコメント記載
- TODOコメントは具体的に記載
